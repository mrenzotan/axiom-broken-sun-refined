## Task 2: Create the `P_SwampParallax` Prefab

**Files:**
- Create: `Assets/Prefabs/Platformer/P_SwampParallax.prefab`
- Create: `Assets/Prefabs/Platformer/P_SwampParallax.prefab.meta`

This task creates the entire prefab in one Unity Editor session, then checks in once at the end. Splitting check-ins per layer would create incomplete intermediate states.

**Layer math summary (referenced by Steps 3–7):**

| Child | Sprite | Draw mode | `m_Size` (units) | Parallax factor | Order in Layer |
|---|---|---|---|---|---|
| `Layer_Far` | `1.png` (sky) | Tiled | 200 × 20.25 | **0.05** | -50 |
| `Layer_MidFar` | `2.png` (distant trees) | Tiled | 200 × 20.25 | **0.2** | -40 |
| `Layer_Mid` | `3.png` (close trees) | Tiled | 200 × 20.25 | **0.4** | -30 |
| `Layer_MidNear` | `4.png` (water + islands) | Tiled | 200 × 20.25 | **0.6** | -20 |
| `Layer_Near` | `5.png` (foreground bushes) | Tiled | 200 × 20.25 | **0.85** | -10 |

**Why 200 × 20.25:**
- `m_Size.x = 200` matches the Snow / Cave / Volcanic convention. Multiplied by root scale 2 → 400 world units of horizontal coverage. Even at the slowest factor (0.05), this covers a level up to ~420 world units long without re-tiling at the level edges.
- `m_Size.y = 20.25` exactly matches the source texture's native height (324 px ÷ PPU 16). Setting Y to native height prevents Unity from tiling vertically — one row of sprite, repeated horizontally only, which is what every layer wants. (`20` would crop ~4 px off the top; the precise `20.25` preserves the full image.) ⚠️ Volcanic / Snow / Cave prefabs all use integer `m_Size.y` because their source heights happened to divide cleanly by PPU 16. Unity accepts the fractional value (`m_Size` is a `Vector2`), but if you'd rather match the integer convention, use `20` and accept the 4 px crop — for `1.png` (flat sky) and `4.png`/`5.png` (top portion is transparent) the crop is invisible. Trees in `2.png`/`3.png` go to the top of the image; cropping there will trim a few pixels of canopy.

**Why these specific factors (0.05 / 0.2 / 0.4 / 0.6 / 0.85):** five-layer extension of the validated Level_1-1 spread (0.1 / 0.5 / 0.85). Sky pushed slightly farther than `2.png`; foreground unchanged from the existing biome convention. Designers can override per-scene if a particular level wants a different feel.

**Why Tiled (not Simple) on every layer:** all five swamp sprites are designed to repeat horizontally — including the foreground (`5.png`) where the bush silhouette and log pattern recurs cleanly across tile seams. Tiled keeps every layer's coverage inexpensive (one SpriteRenderer + native Unity tiling, no scripts) regardless of how wide the host scene becomes.

- [ ] **Step 3: Verify parallax differential rates**

  > **Unity Editor task (user):**
  > Click **Play**. Walk the player left and right. Visually confirm:
  > - `1.png` (sky) barely moves relative to the player.
  > - `2.png` (distant trees) moves slowly — noticeably faster than the sky.
  > - `3.png` (close trees) moves moderately.
  > - `4.png` (water + islands) moves faster than the close trees.
  > - `5.png` (foreground bushes) scrolls close to (but not quite at) the player's speed.
  >
  > If any layer scrolls in the **opposite** direction or scrolls noticeably faster than the player, the parallax factor on that layer is wrong (likely inverted from the spec's 0.05 / 0.2 / 0.4 / 0.6 / 0.85). Open the prefab and re-check the table in Task 2 above.

- [ ] **Step 4: Verify no visible seams in the tiled layers**

  > **Unity Editor task (user):**
  > Continue walking left or right past the level's natural width, watching all five layers. Sprite tiling should be invisible — the bush border, water islands, and tree silhouettes should appear to repeat seamlessly.
  >
  > If a vertical seam (a hard line where the sprite repeats) is visible in `2.png`–`5.png`, the source artwork doesn't tile cleanly at this resolution. **Fallback:** exit Play mode, open the prefab, change the affected layer's Sprite Renderer **Draw Mode** from `Tiled` back to `Simple`, and increase its scale so one un-tiled instance covers the host scene width. Re-check in the prefab after the change.
  >
  > A seam in `1.png` (the flat sky) means `Filter Mode` was left as `Bilinear` — re-verify Task 1 settings on `1.png`.

- [ ] **Step 5: Verify gameplay layers render in front of parallax**

  > **Unity Editor task (user):**
  > Confirm the player sprite, tilemaps, hazards, and any UI render in **front** of all parallax layers (the parallax should always be visually behind everything else). If the player or platforms appear behind the parallax, a sorting layer mismatch exists — re-check that all 5 parallax layer Sprite Renderers use the project's background sorting layer (the one Snow / Cave / Volcanic use), not the `Default` sorting layer.

- [ ] **Step 6: Exit Play mode and clean up the smoke-test instance**

  > **Unity Editor task (user):**
  > 1. Click **Play** again to stop.
  > 2. In the Hierarchy, right-click the `P_SwampParallax` instance → **Delete**.
  > 3. Close `Platformer.unity` without saving (File → Close Scene → Don't Save). Do **not** save any changes prompted by Play mode.
  > 4. UVCS Pending Changes should now show **no scene-related entries**. If `Platformer.unity` appears as modified, discard those changes — the smoke test should leave zero scene-level footprint.

## Execution Notes

- **Estimated total time:** ~25–35 minutes — five sprite-import passes are the slow part; the prefab itself is fast once the import is clean.
- **Prefab authoring requires a scene with a `MainCamera`** — `Platformer.unity` works.
- **Scene-instance overrides on parallax factors are NOT expected.** The prefab carries the correct defaults — drop it into a future swamp scene at the desired position and don't override factors per-instance unless that level's pacing demands it.
- **No swamp-themed scene exists yet.** This plan deliberately stops at "prefab + smoke test." When a swamp level is created (e.g. a future World 3 scene), drop the prefab in as a top-level Hierarchy entry, mirroring how Level_2-1 uses the volcanic prefab.

# Volcanic Area Parallax Background Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **NOTE — no C# scripts in this plan.** Every implementation step is a Unity Editor action performed by the user. The standard TDD pattern (red-green-refactor) does not apply. Verification is via Play mode walkthrough + the existing `ParallaxBackgroundTests.cs` continuing to pass.

**Goal:** Build a reusable `P_VolcanicAreaParallax` prefab from the Volcanic Area asset pack and replace the stub Snow Mountain Parallax in `Level_2-1.unity`.

**Architecture:** Mirror the existing biome prefab pattern — a root GameObject (scale 2) with 4 child layers each carrying a `SpriteRenderer` (Tiled draw mode for the three backdrop layers; Simple for the vapor children) + a `ParallaxController` component. No new scripts. Parallax factors authored with correct defaults (low = far, high = near) so scenes need no per-instance overrides.

**Tech Stack:** Unity 6 LTS, URP 2D, existing `ParallaxController.cs` + `ParallaxBackground.cs`, Unity Version Control (UVCS) for check-ins.

**Spec:** [docs/superpowers/specs/2026-04-30-volcanic-area-parallax-background-design.md](docs/superpowers/specs/2026-04-30-volcanic-area-parallax-background-design.md)

## Task 2: Create the `P_VolcanicAreaParallax` Prefab

**Files:**
- Create: `Assets/Prefabs/Platformer/P_VolcanicAreaParallax.prefab`
- Create: `Assets/Prefabs/Platformer/P_VolcanicAreaParallax.prefab.meta`

This task creates the entire prefab in one Unity Editor session, then checks in once at the end. The prefab is a single atomic unit; splitting check-ins per layer would create incomplete intermediate states.

- [ ] **Step 1: Open a scratch scene to author the prefab in**

  > **Unity Editor task (user):**
  > Open `Assets/Scenes/Platformer.unity` (or any scene with a Camera tagged `MainCamera`). The `ParallaxController` script's `Start()` method requires `Camera.main` to be present in the scene — if you author the prefab in an empty scene with no MainCamera, you will see runtime errors when entering Play mode (though the prefab itself will save fine).

- [ ] **Step 2: Create the prefab root**

  > **Unity Editor task (user):**
  > 1. In the Hierarchy, right-click → **Create Empty**. Rename the new GameObject to `P_VolcanicAreaParallax`.
  > 2. With it selected, in the Inspector set the Transform values:
  >    - Position: `(0, 0, 0)`
  >    - Rotation: `(0, 0, 0)`
  >    - Scale: `(2, 2, 1)` — matches Snow / Cave biome convention.
  > 3. Drag this GameObject from the Hierarchy into `Assets/Prefabs/Platformer/` in the Project window. Unity will create `P_VolcanicAreaParallax.prefab`.
  > 4. The Hierarchy entry should now display in blue text indicating it's a prefab instance.
  > 5. Click the prefab in the Project window and click **Open** (or double-click) to enter Prefab Edit Mode for the rest of this task.

- [ ] **Step 3: Add `Layer_Far` (sky backdrop using `back.png`)**

  > **Unity Editor task (user):**
  > Inside Prefab Edit Mode for `P_VolcanicAreaParallax`:
  > 1. Right-click the root → **Create Empty Child**. Rename to `Layer_Far`.
  > 2. With `Layer_Far` selected, set Transform: Position `(0, 0, 0)`, Rotation `(0, 0, 0)`, Scale `(1, 1, 1)`.
  > 3. Inspector → **Add Component** → **Sprite Renderer**.
  >    - **Sprite:** drag `back.png` from Project window into the Sprite slot.
  >    - **Material:** click the dot picker → select `Sprite-Lit-Default` (this matches Snow / Cave; same material GUID `a97c105638bdf8b4a8650670310a4cd3` if you want to verify by inspecting the .prefab YAML afterward).
  >    - **Sorting Layer:** select the project's background sorting layer (the one Snow / Cave use; should be the layer with ID `1001`, typically named "Background" or similar — confirm by checking what `P_SnowMountainParallax` uses).
  >    - **Order in Layer:** `-40`
  >    - **Draw Mode:** `Tiled`
  >    - **Size:** `X = 200`, `Y = 17`
  > 4. Inspector → **Add Component** → search for `ParallaxController`. Set:
  >    - **Parallax Factor:** `0.05`

- [ ] **Step 4: Add `Layer_MidFar` (`far-mountains.png`)**

  > **Unity Editor task (user):**
  > Still inside Prefab Edit Mode:
  > 1. Right-click the root → **Create Empty Child**. Rename to `Layer_MidFar`.
  > 2. Transform: Position `(0, 0, 0)`, Rotation `(0, 0, 0)`, Scale `(1, 1, 1)`.
  > 3. Add **Sprite Renderer** component:
  >    - **Sprite:** `far-mountains.png`
  >    - **Material:** `Sprite-Lit-Default` (same as Layer_Far)
  >    - **Sorting Layer:** same background layer as Layer_Far
  >    - **Order in Layer:** `-30`
  >    - **Draw Mode:** `Tiled`
  >    - **Size:** `X = 200`, `Y = 17`
  > 4. Add **ParallaxController** component:
  >    - **Parallax Factor:** `0.25`

- [ ] **Step 5: Add `Layer_MidNear` empty container (vapor parent)**

  > **Unity Editor task (user):**
  > 1. Right-click the root → **Create Empty Child**. Rename to `Layer_MidNear`.
  > 2. Transform: Position `(0, 0, 0)`, Rotation `(0, 0, 0)`, Scale `(1, 1, 1)`.
  > 3. **Do NOT add a Sprite Renderer** — this layer is an empty container; only its children render. Vapor plumes (added in Steps 7–8) are intentionally placed BEHIND the volcanos so the volcano silhouettes occlude the vapor bases (matches the artist demo composition: vapor rises from behind the foreground volcanos).

- [ ] **Step 6: Add `Layer_Near` (`volcanos.png`)**

  > **Unity Editor task (user):**
  > 1. Right-click the root → **Create Empty Child**. Rename to `Layer_Near`.
  > 2. Transform: Position `(0, 0, 0)`, Rotation `(0, 0, 0)`, Scale `(1, 1, 1)`.
  > 3. Add **Sprite Renderer** component:
  >    - **Sprite:** `volcanos.png`
  >    - **Material:** `Sprite-Lit-Default`
  >    - **Sorting Layer:** same background layer
  >    - **Order in Layer:** `-10`
  >    - **Draw Mode:** `Tiled`
  >    - **Size:** `X = 200`, `Y = 17`
  > 4. Add **ParallaxController** component:
  >    - **Parallax Factor:** `0.8`

- [ ] **Step 7: Add `VaporPlume_A` child under `Layer_MidNear`**

  > **Unity Editor task (user):**
  > 1. Right-click `Layer_MidNear` → **Create Empty Child**. Rename to `VaporPlume_A`.
  > 2. Transform: Position `(-20, 6, 0)`, Rotation `(0, 0, 0)`, Scale `(1, 1, 1)`.

  >    - **Sprite:** `vapor-a.png`

**Files:** none modified — verification only.

This task confirms the work meets the spec's acceptance criteria. No commits.

- [ ] **Step 1: Run existing Edit Mode tests**

  > **Unity Editor task (user):**

  > 2. In the Test Runner, select the **EditMode** tab.
  > 3. Click **Run All**.
  > 4. Confirm `ParallaxBackgroundTests` (under [Assets/Tests/Editor/SceneSetup/ParallaxBackgroundTests.cs](Assets/Tests/Editor/SceneSetup/ParallaxBackgroundTests.cs)) all pass green. They should — the math class is unchanged.
  >
  > Expected: all parallax tests PASS. If any fail, the math class was unintentionally touched — investigate before continuing.

- [ ] **Step 2: Enter Play mode in `Level_2-1`**

  > **Unity Editor task (user):**
  > With `Level_2-1.unity` open, click **Play** in the Editor toolbar. Wait for the player character to appear in the level's start area.

- [ ] **Step 3: Verify parallax differential rates**

  > **Unity Editor task (user):**
  > Walk the player left and right (using whatever input the level uses — typically arrow keys / A-D / left-stick). Visually confirm:
  > - `back` (sky) barely moves relative to the player.
  > - `far-mountains` moves slowly — noticeably faster than the sky but slower than the foreground.
  > - Vapor plumes move at a moderate rate (behind the volcanos).
  > - `volcanos` scrolls close to (but not quite at) the player's speed.
  >
  > If any layer scrolls in the **opposite** direction or scrolls noticeably faster than the player, the parallax factor on that layer is wrong (likely inverted from the spec's 0.05 / 0.25 / 0.5 / 0.8). Open the prefab and re-check Step 3–6 of Task 2.

- [ ] **Step 4: Verify no visible seams in the tiled layers**

  > **Unity Editor task (user):**
  > Continue walking left or right past the level's natural width, watching the `back`, `far-mountains`, and `volcanos` layers carefully. Sprite tiling should be invisible — sprites should appear to repeat seamlessly.
  >
  > If a vertical seam (a hard line where the sprite repeats) is visible in `far-mountains` or `volcanos`, the source artwork doesn't tile cleanly. **Fallback:** exit Play mode, open the prefab, change the affected layer's Sprite Renderer **Draw Mode** from `Tiled` back to `Simple`, and increase its scale so one untiled instance covers the level width. Re-check in the affected prefab if you make this change.
  >
  > A seam in `back` is a different kind of issue — `back.png` is 48px wide and meant to tile; if a seam is visible, it likely means `Filter Mode` is set to `Bilinear` instead of `Point`. Re-verify Task 1 Step 1 settings.

- [ ] **Step 5: Verify vapor positions match the artist demo**

  > **Unity Editor task (user):**
  > With Play mode still active, visually compare the vapor plume positions to the artist's reference image (one plume left of center, one right; plume bases sitting behind the mountain silhouettes — not floating in the sky and not buried in the ground).
  >
  > If positions look off, exit Play mode, open the prefab, adjust `VaporPlume_A` and/or `VaporPlume_B` local Position Y values inside Prefab Edit Mode, save, then re-enter Play mode to verify. Re-check in the prefab if changed.

- [ ] **Step 6: Verify gameplay layers render in front of parallax**

  > **Unity Editor task (user):**
  > Confirm the player sprite, tilemaps, hazards, and any UI render in **front** of all parallax layers (the parallax should always be visually behind everything else). If the player or platforms appear behind the parallax, a sorting layer mismatch exists — re-check that all 4 parallax layer Sprite Renderers use the project's background sorting layer (the one Snow / Cave use), not the `Default` sorting layer.

- [ ] **Step 7: Exit Play mode**

  > **Unity Editor task (user):**
  > Click **Play** again in the Editor toolbar to stop. Do not save any changes prompted by Play mode (Unity sometimes prompts about transient runtime state — discard these).

## Execution Notes

- **Estimated total time:** ~30–45 minutes if texture imports are clean on first try; longer if vapor positions need tuning by eye against the artist demo.
- **Prefab authoring requires a scene with a `MainCamera`** — `Platformer.unity` works. Don't author the prefab in an empty scene.
